# 账号矩阵维护

## 这篇文档解决什么问题

账号矩阵用于集中维护平台运营账号、平台绑定关系、独立浏览器 profile、登录状态和账号备注。它的目标是让每个账号都具备清晰的身份、独立运行环境和可追踪的状态。

## 适用对象

- 超级管理员：创建账号、调整账号状态、处理账号分配策略。
- 发布员：确认账号是否可用于发布。
- 素材维护员：确认账号内容方向，避免素材投放到错误账号。

## 账号创建规则

创建账号时需要填写：

- 品牌前缀：用于统一账号命名。
- 名称：账号的业务识别名称。
- 账号编号：可自动生成，也可以选择固定编号。
- 内容方向：说明该账号主要发布什么类型内容。
- 账号状态：启用、养号或暂停。

平台会根据账号信息创建账号记录，并为对应平台准备独立运行环境。

## 平台绑定与浏览器 profile

每个账号可以绑定多个平台。平台绑定后通常会关联：

- 平台名称。
- 登录状态。
- 独立浏览器 profile 目录。
- 调试端口。
- 浏览器指纹配置。

独立 profile 的作用是隔离不同账号的登录态、缓存、Cookie 和浏览器环境，降低账号互相污染的风险。

## 登录状态处理

常见登录状态包括：

- unknown：尚未检测。
- valid：登录可用。
- expired：登录失效，需要重新扫码或重新登录。
- checking：正在检测。

如果发布任务失败并提示登录失效，先在账号矩阵中确认该账号平台的登录状态，再进入通知中心或任务中心处理后续动作。

## 账号状态建议

- 启用：可以参与发布和任务分配。
- 养号：账号存在，但不建议参与高频发布。
- 暂停：暂时不参与发布，适合异常账号、待整改账号或临时停用账号。

## 常见问题

### 为什么创建账号后没有立即发布？

账号创建只代表档案和运行环境建立完成，不代表平台登录已完成。需要先完成平台登录，并确认公共设置中的发布策略。

### 为什么同一个账号要使用独立浏览器？

独立浏览器 profile 可以隔离登录状态和平台缓存，避免多个账号共用同一浏览器环境导致平台识别混乱。

### 删除账号前要注意什么？

确认该账号没有正在运行的任务、没有待处理登录提醒，并记录需要保留的发布结果或素材信息。

## 操作检查清单

- 账号命名是否清晰。
- 内容方向是否准确。
- 平台绑定是否完整。
- 登录状态是否可用。
- 暂停账号是否已从发布计划中排除。
