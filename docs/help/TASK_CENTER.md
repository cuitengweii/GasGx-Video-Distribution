# 发布任务与队列

## 这篇文档解决什么问题

任务中心用于查看发布、评论、私信、统计和登录检测等任务的执行状态。它帮助运营人员判断任务是否排队、运行、成功、失败或需要重试。

## 适用对象

- 发布员：查看发布任务进度和失败原因。
- 超级管理员：排查任务调度、队列堆积和账号异常。
- 数据监控员：确认统计任务是否完成。

## 任务类型

常见任务包括：

- 自动发布：把生成好的视频发布到目标平台。
- 自动评论：按策略执行评论互动。
- 自动私信：按线索或运营计划发送私信。
- 数据统计：采集播放、互动、转化等数据。
- 登录检测：检查账号平台登录状态。

## 任务状态

- pending：任务已创建，等待执行。
- running：任务正在执行。
- success：任务完成。
- failed：任务失败，需要查看错误信息。
- retrying：任务正在重试。
- canceled：任务被取消或删除。

## 排查任务失败

1. 查看任务类型和目标平台。
2. 查看账号是否可用。
3. 查看错误摘要和详细错误。
4. 如果是登录失效，先处理登录提醒。
5. 如果是素材或文件错误，先检查素材目录。
6. 如果是平台页面变化，记录错误并暂停同类批量任务。

## 删除任务前要确认

删除任务只应处理确认不再需要的任务。删除前确认：

- 任务不是正在运行的关键发布任务。
- 失败原因已经记录。
- 不需要保留该任务作为排查证据。

## 常见问题

### 为什么任务一直 pending？

可能是后台服务未运行、队列已有长任务占用、定时策略未触发，或账号不符合当前发布条件。

### 为什么任务显示成功但平台没看到内容？

可能是平台保存为草稿、发布审核延迟、账号发布入口变化，或平台页面状态识别不完整。进入对应平台确认最终结果。

### 为什么失败任务重复出现？

检查是否开启失败补跑，或是否有定时任务反复触发同一个失败条件。

## 操作检查清单

- 发布前清理历史失败任务。
- 运行中不要关闭本地服务。
- 失败后先读错误摘要再重试。
- 批量失败时暂停同类任务，避免扩大影响。
