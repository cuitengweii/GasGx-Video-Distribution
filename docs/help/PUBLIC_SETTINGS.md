# 公共设置与平台配置

## 这篇文档解决什么问题

公共设置用于配置发布素材目录、默认发布策略、平台参数和矩阵发布作业规则。它决定平台如何读取素材、如何分配账号、什么时候启动发布，以及失败后是否补跑。

## 适用对象

- 超级管理员：维护全局策略、平台参数和定时发布规则。
- 发布员：检查发布前的素材目录、发布方式、批次和间隔。
- 素材维护员：确认素材目录是否指向正确位置。

## 全局公共配置

全局公共配置会被各平台默认继承：

- 发布素材目录：平台读取待发布视频或素材的目录。
- 默认发布方式：立即发布或保存草稿。
- 全局话题：发布文案默认携带的话题。
- 上传超时：单个平台上传等待的最长时间。

如果平台没有单独配置，系统会使用全局公共配置作为默认值。

## 平台独立配置

平台独立配置优先级高于全局配置。适合处理不同平台规则差异，例如：

- 平台文案格式。
- 可见范围。
- 评论权限。
- 视频号合集名称。
- 原创声明。
- 短标题和地理位置。

配置优先级为：平台独立配置高于全局公共配置，高于代码默认值。

## 矩阵发布作业

矩阵发布作业控制账号批次、启动方式和失败补跑：

- 每批账号数：一次发布批次中参与的账号数量。
- 后台定时启动：是否由后台调度自动启动。
- 定时模式：按间隔或每天固定时间。
- 间隔发布：两次调度之间的时间。
- 批次最小/最大间隔：批次内部的间隔范围。
- 起始批次轮换：避免每次都从同一组账号开始。
- 批次内随机顺序：降低固定顺序带来的风险。
- 失败最后补跑：把失败账号放到最后重试。

## 发布前检查流程

1. 确认素材目录存在且有可用视频。
2. 确认默认发布方式符合当天策略。
3. 检查全局话题是否需要更新。
4. 检查平台独立配置是否覆盖了全局值。
5. 检查矩阵发布作业是否处于预期模式。
6. 手动启动前确认弹窗里的提示，再执行发布。

## 常见问题

### 为什么平台配置和全局配置不一致？

平台独立配置优先级更高。如果某个平台设置了独立值，发布时会使用平台值，而不是全局值。

### 为什么定时任务没有启动？

检查后台定时启动是否开启、定时模式是否正确、服务是否持续运行，以及上次任务是否仍在执行。

### 为什么失败账号没有马上重试？

如果开启“失败最后补跑”，失败账号会被放到发布链路后段，避免阻塞当前批次。

## 操作检查清单

- 素材目录正确。
- 发布方式正确。
- 平台覆盖项已确认。
- 定时模式符合当天计划。
- 批次和间隔符合账号安全策略。
