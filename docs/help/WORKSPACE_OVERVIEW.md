# 工作台总览

## 这篇文档解决什么问题

工作台总览是运营人员每天进入平台后的第一站，用来判断今天是否可以继续生成、发布和复盘内容。它不替代各功能页的详细操作，而是提供一个快速体检入口：账号是否可用、素材是否足够、任务是否堆积、发布是否失败、通知是否需要处理。

## 适用对象

- 超级管理员：确认平台整体状态，决定是否需要调整账号、权限或系统配置。
- 发布员：查看今日发布任务、失败任务和待处理提醒。
- 素材维护员：确认素材余量、视频生成入口和素材相关提醒。
- 数据监控员：从总览进入数据统计与运营复盘。

## 页面主要区域

### 顶部指标

顶部指标用于快速判断当前运营负载：

- 账号：当前平台已维护的矩阵账号数量。
- 平台槽位：所有账号绑定的平台入口数量。
- 剩余素材：可用于生成或发布的素材余量。
- 运行中任务：正在执行或排队的发布、生成、检测任务。
- 失败任务：需要优先排查的异常任务。
- 播放、评论：近期运营效果的基础数据入口。

### 今日运营重点

今日运营重点把最常用的三个动作集中在首页：

- 登录、素材、发布异常：优先处理会阻断发布的事项。
- 视频生成：进入批量生成工作台，处理封面、正文模板、片尾、背景音乐和素材组合。
- 执行队列：查看正在运行、失败、待重试的任务。

### 常用入口

常用入口按运营动线排列：

- 账号矩阵：维护账号资料、平台登录状态和浏览器 profile。
- 公共设置：确认素材目录、发布策略和矩阵发布规则。
- 数据统计：复盘播放、互动、线索和转化。
- 帮助文档：查看操作说明、排错流程和知识库。

## 推荐日常流程

1. 先看顶部指标，确认失败任务和剩余素材是否异常。
2. 如果失败任务大于 0，先进入任务中心或通知中心定位原因。
3. 如果剩余素材不足，先补充素材，再进入视频生成。
4. 如果账号或登录状态异常，进入账号矩阵处理。
5. 发布后进入数据统计页查看内容表现。

## 常见问题

### 为什么总览显示任务失败，但任务中心看不到新任务？

可能是旧任务状态未刷新。先点击“刷新当前数据”，再进入任务中心查看。如果仍不一致，检查本地服务是否重启过，或数据库是否切换到了不同品牌实例。

### 为什么快捷入口被隐藏？

入口显示受当前登录角色权限控制。发布员、素材维护员、数据监控员默认只能看到与角色相关的入口。需要超级管理员进入用户中心调整角色权限。

### 为什么总览数字为空？

通常是数据库尚未初始化、当前账号没有数据，或服务刚启动未完成加载。进入系统设置执行初始化检查，再刷新当前数据。

## 操作检查清单

- 发布前：账号可用、素材充足、失败任务已处理。
- 生成前：素材分类完整、模板已保存、输出目录可写。
- 复盘前：确认统计数据已导入或同步。
- 交接前：把未处理提醒、失败任务和当前发布批次记录清楚。
